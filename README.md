# Face Recognition – ML Project

A **complete, production-ready face recognition system** built with Python and
[`face_recognition`](https://github.com/ageitgey/face_recognition) (backed by
dlib's state-of-the-art deep learning models).

---

## Features

| Capability | Details |
|---|---|
| **Face detection** | HOG (fast, CPU) or CNN (accurate, GPU) models |
| **Face encoding** | 128-dimensional deep embeddings per face |
| **Recognition** | Euclidean-distance matching with configurable tolerance |
| **Training** | Encode a labelled dataset and persist it as a model file |
| **Image inference** | Annotate and save results for static images |
| **Video inference** | Real-time webcam recognition at ~30 FPS |

## Quick Start

### 1 – Install dependencies

```bash
pip install -r requirements.txt
```

> **Note:** `dlib` (required by `face_recognition`) needs CMake and a C++
> compiler.  On Ubuntu/Debian:
> ```bash
> sudo apt install build-essential cmake
> ```
> On macOS: `brew install cmake`

### 2 – Prepare your dataset

Create one sub-directory per person inside `dataset/`:

```
dataset/
├── Alice/
│   ├── alice_01.jpg
│   └── alice_02.jpg
└── Bob/
    └── bob_01.jpg
```

See [`dataset/README.md`](dataset/README.md) for tips on image quality.

### 3 – Train the model

```bash
python train.py
# or via the unified CLI:
python main.py train
```

Options:

```
--dataset          Path to dataset directory (default: dataset/)
--model            Output model path (default: models/face_encodings.pkl)
--detection-model  hog (fast) | cnn (accurate)  [default: hog]
--tolerance        Match threshold 0.0–1.0 (default: 0.6)
-v / --verbose
```

### 4 – Recognise faces in an image

```bash
python recognize_image.py --image photo.jpg --output result.jpg
# or:
python main.py image --image photo.jpg --output result.jpg
```

### 5 – Real-time webcam recognition

```bash
python recognize_video.py
# or:
python main.py video
```

Press **q** to quit.

## API Usage

```python
from src.face_recognizer import FaceRecognizer
from src.utils import load_image, draw_bounding_box, save_image

# Load a pre-trained model
recognizer = FaceRecognizer(model_path="models/face_encodings.pkl")

# Run recognition on an image
rgb_image = load_image("group_photo.jpg")
results = recognizer.recognize(rgb_image)

for name, confidence, bbox in results:
    print(f"{name}: {confidence:.1%}")
    draw_bounding_box(rgb_image, bbox, name, confidence)

save_image(rgb_image, "annotated.jpg")
```

### Add a new person without retraining from scratch

```python
recognizer.add_person("Carol", ["carol_1.jpg", "carol_2.jpg"])
```

## How It Works

```
 Input image
      │
      ▼
 Face Detection (HOG / CNN)
      │  bounding boxes
      ▼
 Face Encoding (dlib ResNet)
      │  128-d embedding
      ▼
 Nearest-neighbour Matching
      │  Euclidean distance
      ▼
 Name + Confidence
```

1. **Detection** – `dlib`'s HOG-based or CNN-based detector locates face
   bounding boxes.
2. **Encoding** – A pre-trained ResNet model (provided by `face_recognition`)
   maps each face crop to a 128-dimensional embedding.
3. **Matching** – The query embedding is compared against all stored embeddings
   using Euclidean distance.  The closest match below the `tolerance` threshold
   wins; otherwise the face is labelled *Unknown*.

## Configuration Reference

| Parameter | Default | Description |
|---|---|---|
| `detection_model` | `"hog"` | `"hog"` = fast CPU, `"cnn"` = accurate GPU |
| `tolerance` | `0.6` | Lower → stricter matching |
| `upsamples` | `1` | Higher → better small-face detection, slower |
| `scale` (video) | `0.5` | Frame downscale factor before detection |
